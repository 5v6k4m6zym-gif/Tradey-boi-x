OBJECTIVE
========================================================

Build a complete:

========================================================
ARCHITECTURE (NON-INTRUSIVE)
========================================================

Existing Flow:
    Model → Signal → Execution

New Flow (parallel system only):
    Model → Signal → Execution
                 ↘

                 ↘
        Backtest + Forward Validator + Analytics

IMPORTANT:
- Audit system MUST NOT affect execution

========================================================
PART 1: BACKTESTING ENGINE (HISTORICAL SIMULATION)

Create BacktestEngine that:

Inputs:
- historical price data
- recorded trade signals OR replayed signals
- strategy parameters (from current bot outputs)

- entry execution
- stop loss hit
- take profit hit
- time-based exits (if applicable)
- slippage model (simple configurable approximation)

Outputs per trade:
- win/loss
- PnL in R multiple
- max favorable excursion (MFE)
- max adverse excursion (MAE)
- duration
- setup type (if available)

Must support:
- batch backtesting over full dataset
- rolling window backtests

========================================================
PART 2: FORWARD VALIDATION ENGINE (SHADOW MODE)

Create ForwardValidator that:

- runs alongside live bot
- records every trade signal decision
- tracks what actually happens after entry
- compares predicted edge vs real outcome

Must log:

- predicted probability
- edge score
- regime type
- actual outcome
- deviation from expectation

IMPORTANT:
- NO live trading changes allowed
- purely observational system

========================================================
PART 3: PERFORMANCE ANALYTICS ENGINE

Create PerformanceAnalytics that calculates:

- win rate overall
- win rate by regime
- expectancy per setup type
- average R per trade
- drawdown curve
- profit factor
- streak analysis (win/loss clusters)

Must support:

- rolling windows (50, 100, 200 trades)
- regime-based breakdown
- signal-quality segmentation (edge score buckets)

========================================================
PART 4: SYSTEM AUDIT ENGINE (CRITICAL)

Create SystemAudit that detects:

A) Logic anomalies:
- unusually high rejection rates
- sudden drop in trade frequency
- inconsistent signal structures

B) Performance anomalies:
- regime mismatch performance drops
- edge score not correlating with outcomes
- calibration drift (probability vs reality mismatch)

C) Data issues:
- missing fields in trades
- NaN or invalid values
- malformed signals

When detected:
- log warning
- do NOT stop system
- suggest likely cause

========================================================
PART 5: BUG DETECTION + SAFE FIX LAYER

Create BugDetector that:

- identifies runtime issues in logs
- detects inconsistent trade objects
- flags broken evaluation outputs

If fix is possible safely:
- suggest patch (DO NOT auto-apply to core system)
- otherwise log only

IMPORTANT:
- NEVER modify production trading logic
- ONLY report + suggest fixes
